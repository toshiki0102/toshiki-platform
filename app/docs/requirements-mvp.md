# 写真ギャラリープラットフォーム 要件定義（MVP）

## 1. プロジェクト概要

個人で収集した写真（風景・ポートレートなど）を一般公開するためのギャラリープラットフォーム。
まずMVPを完成させ、後続フェーズで課金機能を追加する。

---

## 2. ターゲットユーザー

- **閲覧者**: 不特定多数（認証・ログイン不要）
- **管理者**: オーナー本人のみ（写真のアップロード・管理）

---

## 3. 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js (App Router) + React |
| データベース | Supabase (PostgreSQL) |
| ストレージ | Supabase Storage |
| ホスティング | Vercel（推奨） |

---

## 4. MVP 機能要件

### 4.1 閲覧者向け機能

#### ギャラリー一覧ページ（必須）
- 写真をグリッドレイアウトで一覧表示する
- 写真にはサムネイル・タイトル・タグを表示する
- レスポンシブ対応（PC / スマートフォン）

#### タグ・カテゴリ絞り込み（必須）
- タグ一覧をフィルターUIとして表示する
- タグを選択すると該当写真のみ表示される
- 複数タグの複合絞り込みは任意（MVP後でも可）

### 4.2 管理者向け機能（MVP）

- `/admin` ルートに管理画面を実装する
- Google OAuth（Supabase Auth）でログインした管理者のみアクセス可能とする
- `middleware.ts` で未認証ユーザーをログインページにリダイレクトする
- 管理画面から写真をアップロードできる
- アップロード時にタイトル・タグ・カテゴリを入力・登録できる
- アップロードした写真の一覧表示・削除ができる

---

## 5. 認証要件

- **方式**: Supabase Auth の Google OAuth のみ
- **閲覧者**: 認証不要（ギャラリーは一般公開）
- **管理者**: Google アカウントでログインし、管理機能にアクセスする
- メール/パスワード認証・他のSSOは対象外

### 認証フロー

1. 管理者が `/admin` などの保護ルートにアクセスする
2. 未認証の場合、Googleログイン画面にリダイレクトされる
3. 認証成功後、管理画面にリダイレクトされる

### Supabase 設定

- Supabase ダッシュボードで Google プロバイダーを有効化する
- Google Cloud Console で OAuth クライアントIDを発行し、リダイレクトURIを設定する
- RLS（Row Level Security）ポリシーで管理者のみ書き込み可能にする

---

## 6. MVP 対象外（後続フェーズ）

- 課金・有料プラン機能
- お気に入り機能
- 画像詳細ページ（モーダルまたは個別ページ）
- 検索機能（全文検索）
- 管理者の複数アカウント対応

---

## 6. データモデル（Supabase）

### `photos` テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| id | uuid | 主キー |
| title | text | 写真タイトル |
| description | text | 説明文（任意） |
| storage_path | text | Supabase Storage上のパス |
| category | text | カテゴリ（例: 風景、ポートレート） |
| created_at | timestamptz | 登録日時 |

### `tags` テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| id | uuid | 主キー |
| name | text | タグ名（例: 自然、夕焼け） |

### `photo_tags` テーブル（中間テーブル）

| カラム名 | 型 | 説明 |
|---|---|---|
| photo_id | uuid | photos.id への外部キー |
| tag_id | uuid | tags.id への外部キー |

## 7. 推奨ディレクトリ構成（Next.js App Router）

```
/
├── app/
│   ├── page.tsx              # ギャラリー一覧（トップページ）
│   ├── layout.tsx            # 共通レイアウト
│   ├── login/
│   │   └── page.tsx          # Googleログインページ
│   └── admin/
│       ├── layout.tsx        # 管理画面レイアウト（認証ガード）
│       ├── page.tsx          # 写真管理一覧
│       └── upload/
│           └── page.tsx      # 写真アップロード
├── components/
│   ├── GalleryGrid.tsx       # 写真グリッド
│   ├── PhotoCard.tsx         # 写真カード（サムネイル）
│   └── TagFilter.tsx         # タグ絞り込みUI
├── lib/
│   └── supabase.ts           # Supabaseクライアント初期化
├── middleware.ts             # 管理ルートの認証ガード
└── types/
    └── index.ts              # 型定義
```

## 9. 非機能要件

- **パフォーマンス**: Next.js の ISR または SSG を用いて初期表示を高速化する
- **画像最適化**: `next/image` コンポーネントを使用し、自動的にリサイズ・WebP変換を行う
- **セキュリティ**: Supabase Storage のバケットポリシーで読み取りのみ公開とする
